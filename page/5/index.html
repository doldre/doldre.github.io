Crazy Rows(GCJ 2009 Round2 A）

Crazy Rows

Problem A. Crazy Rows This contest is open for practice. You can try
every problem as many times as you like, though we won’t keep track of
which problems you solve. Read the Quick-Start Guide to get started.
Small input 6 points Solve A-small Large input 10 points Solve
A-large Problem

You are given an N x N matrix with 0 and 1 values. You can swap any
two adjacent rows of the matrix.

Your goal is to have all the 1 values in the matrix below or on the
main diagonal. That is, for each X where 1 ≤ X ≤ N, there must be no 1
values in row X that are to the right of column X.

Return the minimum number of row swaps you need to achieve the goal.

Input

The first line of input gives the number of cases, T. T test cases
follow. The first line of each test case has one integer, N. Each of
the next N lines contains N characters. Each character is either 0 or
1.

Output

For each test case, output

Case #X: K where X is the test case number, starting from 1, and K is
the minimum number of row swaps needed to have all the 1 values in the
matrix below or on the main diagonal. You are guaranteed that there is
a solution for each test case.

Limits

1 ≤ T ≤ 60 Small dataset

1 ≤ N ≤ 8 Large dataset

1 ≤ N ≤ 40 Sample

Input Output 3 2 10 11 3 001 100 010 4 1110 1100 1100 1000 Case

#1: 0 Case #2: 2 Case #3: 4

思路很简单，把第i行的1最后出现的位置记录下来，记为a[i],如果该行全为0则记为a[i]=-1,按照题意第i行只需要满足a[i] <= i就好了， 所以采取贪心策略，从第i行往后搜索第一个符合条件的a[j],然后把a[j]交换到第i个，这样就可以依次把第一行第二行…确定下来，并且保证交换次数最少。

POJ3255(Dijkstra 求次短路，经典)

发表于 2015-04-17   |  

POJ3255

给出一个节点数为N的无向图，求出从节点1到N的次短路。
Dijkstra的变种用法。 某个点v的次短路，要么是到与之相连的点u的最短路加上u和v的一条边，要么就是到u的次短路加上u和v的一条边生成的。所以讲原来的dijkstra每次只保存最短路不同，只需再增加一个数组用来记录次短路，然后更新次短路的时候依然也加入队列，则可以求出最后N点的次短路和最短路。结果输出次短路就好。

给出两个数S,E，让计算满足{ X的二进制表示中0的个数大于等于1的个数，X属于[S,E]}的X的个数。 记F(X) 为[0,X)中满足
X的二进制表示中0的个数大于等于1的个数。结果等于F(E)-F(S),如果E本身满足此条件，则结果加1。
貌似有人有DP算F(N)，实在不知道DP如何做但是想到了另外一个思路。
如36的二进制为100100，则是100000-100100，10000-100000，
1000-10000,100-1000,10-100,0-10中满足的个数相加，而列入10000-100000中满足条件的个数则与四个二进制位中0的个数大于等于3的组合数相等即=C(4)_4 + C(4)_3，而100000-100100中满足条件的个数，可以看出当第三位1为0时，后面两位无论怎么选都是满足的。这个比较特殊，如100000-101000，当第4位的1为0时，后面三位中0的个数大于等于1都是满足条件的，即C(3)_1+C(3)_2+C(3)_3;

POJ3687

发表于 2015-04-02   |  

神气的拓扑排序，需要使用的优先队列，并且需要反向构图。
这道题我做的时候碰了一鼻子灰。 题目都没有看明白，输出都弄错了。然后看了别人的题解，发现自己的方法都是错的。看到有人说正向构图也能做，我先是尝试了一下，发现还是WA，仔细想了想，如果用优先队列这种思路正向构图是不可以做的。因为尽管每次保证的是从当前队列中选取最小的排到前面，不能保证全局最小的在最前面。正向构图只能保证更大的数尽可能的在后面。反向构图则可以让全局中更小的数在前面，从而保证在符合拓扑排序的条件下，让lable1的weight尽可能小，其次让lable2的weight尽可能小。可以用几个较小的数据规模模拟一下变可以发现这个规律。

修改天平(UVa12166， 二叉树特性)

发表于 2015-04-02   |  

这道题有个很特殊的地方，二叉树如果要平衡，则两个子节点的重量应该相等。并且为上一个的1/2，所以对于每一个平衡的天平，位于相同深度的所有节点的重量应该相等，并且该天平的总重量M与这一层单个节点的重量m关系为，M = m << dep。 这道题就变成了遍历每一个叶的重量，寻找出现的总重量最多的次数，然后用叶节点的数目减去这个次数就是最少需要修改的数目。利用这个特性很巧妙的解决了这个题。
