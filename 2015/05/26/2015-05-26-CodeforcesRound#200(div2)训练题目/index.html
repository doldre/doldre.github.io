  1. Problem C: Rational Resistance

题意：输入a, b两个数，表示一个阻值为a/b电阻，现在有若干阻值为1的电阻，让你用串联或者并联的方式组成一个阻值为a/b的电路。输出需要的单位电阻的最小数量。

思路：可以发现，当a/b < 1的时候，a/b = 1/(1+(b-a)/a), 按照左边等式的思路需要构造a 个阻值为1/b的电阻，这样需要的电阻数目为ab, 按照右边等式需要一个阻值为1的电阻 和 构造出来的一个阻值为a/(b-a)的电阻并联，这样需要的电阻数量为 1 + a(b-a). 然后两式相减得到 ab - (1+a(b-a)) = a^2 -1。 因为a > 0, 所以对于任意a> 1左式需要的电阻数是大于右式的。所以本题的思路就是一直按照此等式进行递归分解，直到a=1。
但是注意到这是a/b < 1的情况，但是a/(b-a)是有可能大于>=1的。 很明显大于1的情况分解为a/b = k + p/q（带分数) 这种k个单位电阻串联然后和p/q的电路串联所需要的单位电阻最小，然后p/q按照 a/b < 1的情况处理即可。所以这道题用递归可以很优雅的解决。

PS：其实这道题，我最开始的思路是没有推导到a=1的时候最小的，使用了一个偷懒的方法，每一次递归的时候我是取得min(ab, 1 + f(a, b - a))，然后一直在第27组数据wa了，一直想不明白原因（这么优雅的思路不应该wa掉啊！！！！)， 最后比赛结束过后看到代码，差点剁手了， 因为题目给出的a，b最大可能是10^18, 所以ab的过程中超出了long long 的范围导致计算溢出了，然后就这样华丽丽的wa掉了。所以以后进行乘法操作的时候，要特别小心是否会溢出，如果有可能溢出的话最好想其他等效做法，或者做特殊处理。
